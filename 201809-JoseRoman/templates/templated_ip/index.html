Internet Protocol version 4 IPv4

Protocolo de la capa de RED

El protocolo IP es uno de los protocolos más famosos que existe. Internet es una puerta al mundo y un recurso que la mayoría de las personas utiliza de forma diaria, y el responsable de que se transmita la información es, en gran medida, gracias a este protocolo.

Si bien es cierto que este protocolo no se ve en "Arquitectura de Redes" sino, en la asignatura de "Redes de Computadores", es interesante añadirlo para ver como se tranducen las conexiones TCP y UDP a IP.

Best-Effort Protocol

Una de las características más llamativas de IP es que no garantiza que la información que envías llegue a su destino. El camino que tiene que realizar el datagrama IP desde el origen hasta el destino puede estar, congestionado, cortado, estropeado, encerrado en un bucle, etc... Un protocolo "Best-Effort" hace el "mayor esfuerzo" posible para que el datagrama llegue al destino, pero no lo garantiza. Por eso, protocolos de capas superiores como TCP tienen mecanismos de recuperación frente a errores, porque como en este caso, algunas veces los protocolos de capas inferiores no los implementan.

La dirección IPv4

La dirección IP es un dato de 32bits, es decir, 4 bytes, el cual permite el uso de 4294967296 direcciones. Aunque parezcan muchas direcciones, lo cierto es que se están acabando, por ello se creó el protocolo IPv6, cuyas direcciones tienen 128 bits, es decir, es como 4 direcciones IP juntas.

Características de las direcciones IP

  • Rangos de direcciones IP

    No todas las direcciones IP son válidas o se pueden utilizar. Hay rangos de IPs que utiliza el propio sistema operativo y existen otras direcciones que están reservadas. Las direcciones válidas se agrupan en 3 clases A,B y C. Estas clases se identifican por los bits iniciales de la dirección y por el tamaño de la red que representan. Este concepto se estudia con más detenimiento en la asignatura Redes de Computadores.

  • Fragmentación y reensamblado

    No todos los caminos entre routers son iguales. La Unidad Máxima de Transferencia(MTU), determina la cantidad de datos que se pueden transmitir por la red y por lo tanto, condiciona el tamaño de los datagramas IP. Cuando un datagrama se encuentra en un router y la MTU es menor que el tamaño del paquete, el paquete se fragmenta y se envía, junto con un identificador que determina el orden en el que los paquetes deben ser reensamblados.

  • TTL o Tiempo de Vida(Time To Live)

    En la red se pueden crear bucles debido a errores, fallos en la configuración e incluso, ataques. Si se crea un bucle, el datagrama se queda encerrado en el bucle viajando entre los mismos routers. Esto lleva a la congestión de la red y a la consecuente perdida de información. Para evitar esto, los paquetes IP tienen un contador llamado TTL. Este contador se decrementa en cada salto que realiza en la red, es decir, por cada router que pasa. Si el TTL llega a 0, el paquete es descartado.

    ¿Cómo se consiguen las direcciones IP?

  • DHCP Dynamic Host Configuration Protocol

    Cuando un nuevo equipo se conecta a cualquier red, lo primero que hace es buscar el servidor DHCP para que este le asigne una dirección IP. Los routers domésticos tienen servidores DHCP los cuales asignan las direcciones en la red interna doméstica, en entornos empresariales, estos servidores DHCP se instalan en servidores Windows o Linux.

  • NAT Network Address Translation

    Para evitar un sobreuso de direcciones IP, lo que se utiliza actualmente es el protocolo NAT. Como ejemplo de utilización de NAT, tenemos la distribución de direcciones IP en nuestras redes domésticas. La dirección IP de nuestros equipos nos la proporciona el router. De cara a Internet, el router solo tiene una sola IP, pero el tráfico sale por diferentes puertos. Cada uno de esos puertos por los que sale y entra información en nuestro router, es un flujo de datos que va y viene a cada uno de nuestros dispositivos. El router tiene una tabla de traducción que le dice, en base al puerto al que llega la información, a que dispositivo le corresponde.
