HTTP Capa de Aplicación

Hypert Text Transfer Protocol HTTP

Protocolo de la capa de aplicación

HTTP es uno de los protocolos más importantes que existen debido a que es el principal responsable de que puedas navegar por la red. Te sonarán sus siglas, ya que en la mayoría de los navegadores suele aparecer a la izquierda de la URL de la página web en la que te encuentres. Como pequeño recordatorio, recuerda que las páginas web son descargadas en tu navegador y el resultado que tu ves es el archivo html junto con otra información adjunta a la web que renderiza tu navegador.

En pocas palabras, HTTP es el responsable de que te llegue la página web a tu dispositivo, es decir, que el archivo HTML y sus objetos referenciados lleguen a tu navegador.La comunicación la realizan un cliente y servidor, el primero enviando solicitudes, y el segundo respondiendo con la información pedida o devolviendo un error.

Protocolo sin estado

HTTP no tiene estados, esto quiere decir que no guarda información del contexto de la comunicación entre un servidor y el cliente. Que HTTP sea un protocolo sin estado significa que, en la implementación inicial del protocolo, el servidor no recuerda el cliente. Sin embargo, existen métodos para guardar una cierta información del cliente, por ejemplo, mediante cookies, claves almacenadas en el navegador. Con ellas, el servidor puede recuperar cierta información y modificar su respuesta en base a lo que esté  guardado.

Usa TCP como protocolo de transporte

HTTP utiliza TCP como protocolo de la capa de transporte, esto quiere decir que si la información llega a su destino, esta llegará correctamente y completa. A diferencia de servicios como el streaming de video, en el cual se puede perder la información de un pixel (o más), en protocolos como HTTP que toda la información llegue correcta y completa es una de las principales prioridades.

Persistente VS No Persistente

La diferencia depende principalmente de la cantidad de objetos enviados en una conexión TCP y como se ha implementado.

Persistente

Se envían varios objetos mediante una conexión TCP. Si la página web esta formada por 3 objetos, se abre una conexión TCP, se envía el primer objeto, se envía el segundo, y así hasta que se han enviado todos los objetos.

No Persistente

Solo se envía un objeto por cada conexión TCP. Es decir , se abre una conexión TCP, se envía un objeto (pagina web, imágen de la página, etc...), se cierra la conexión , y así hasta que se han enviado todos los objetos que conforman la página web.
